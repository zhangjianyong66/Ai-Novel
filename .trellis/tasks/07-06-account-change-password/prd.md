# 登录用户修改自己的密码

## Goal

让已登录用户可以在前端自助修改自己的本地登录密码，避免只能依赖管理员重置密码。

用户价值：

- 普通用户和管理员都可以在知道旧密码的情况下自行更新密码。
- 管理员仍保留用户管理页的重置密码能力，但自助修改密码不应依赖管理员权限。

## Confirmed Facts

- 后端已有普通改密接口 `POST /api/auth/password/change`，入参为 `old_password` 和 `new_password`。
- 后端改密接口要求当前已登录用户，校验旧密码，成功后只更新密码哈希和更新时间。
- 后端现有测试已覆盖旧密码失效、新密码可登录。
- 前端当前没有普通用户自助修改密码入口；`AuthContext` 只暴露登录、注册、退出和刷新。
- 前端当前已有账户级路由 `/account/notification-settings`，管理员用户管理页已有管理员重置密码能力。

## Requirements

- 新增登录用户可访问的账户安全页，路由使用 `/account/security`。
- 账户安全页提供修改密码表单，字段为“当前密码 / 新密码 / 确认新密码”。
- 表单在前端校验新密码至少 8 位，且两次输入一致。
- 提交时调用当前登录用户改密接口，使用旧密码和新密码。
- 修改成功后不强制退出当前会话，展示成功提示并清空表单。
- 旧密码错误、未登录、账号没有本地密码等失败场景应向用户展示可理解的错误提示。
- 导航或用户区域应提供进入账户安全页的入口。
- 管理员用户也通过同一自助入口修改自己的密码；管理员重置他人密码能力保持不变。

## Acceptance Criteria

- [x] 已登录本地密码用户可以打开 `/account/security` 并看到三字段修改密码表单。
- [x] 新密码少于 8 位时，前端阻止提交并提示原因。
- [x] 两次新密码不一致时，前端阻止提交并提示原因。
- [x] 输入正确旧密码后提交成功，页面提示密码已修改，当前登录会话继续可用，表单被清空。
- [x] 输入错误旧密码时，页面显示后端返回的“旧密码错误”等错误信息。
- [x] 管理员登录后同样可以使用账户安全页修改自己的密码。
- [x] 用户管理页的管理员重置密码功能不被本次改动破坏。

## Out of Scope

- 不实现第三方登录账号首次设置本地密码。
- 不实现修改密码后踢出当前会话或所有其他会话。
- 不调整后端密码策略，沿用现有最小长度和哈希逻辑。
