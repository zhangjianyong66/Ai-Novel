# 前端移动端兼容优化设计

## 目标边界

本任务不重写前端信息架构，而是在现有 React/Tailwind 风格与组件拆分基础上补齐移动端可用性。核心原则是手机竖屏可完成创作闭环，桌面端保持现有工作台效率。

## 全局布局

- `AppShell` 继续作为全局布局边界。
- 桌面端保留现有侧边栏折叠逻辑。
- 移动端复用现有导航抽屉方向，补齐宽度、高度、滚动、点击关闭、焦点和遮挡问题。
- 页面容器需要保证 `min-w-0`、合理 `px`、`max-width` 和内容换行，避免子级撑出 viewport。

## 通用 UI

- `Drawer` 在手机端默认接近全屏，内容区可垂直滚动，底部操作区需要考虑安全区和固定按钮遮挡。
- `Modal` 在手机端应使用窄屏友好的宽度、高度和滚动策略，避免标题、按钮或正文被裁切。
- 按钮组在手机端允许换行或纵向排列；关键主操作保持可见。
- 表单在手机端优先单列布局，长 label、长模型名、长错误文本必须换行或截断。

## 首批页面

### AppShell

- 保留“抽屉导航，不做底部 Tab”的方案。
- 手机顶部保留菜单按钮、当前页面标题、登录/登出和主题切换入口。
- 移动端抽屉展示完整导航分组，点击导航项后关闭抽屉。

### DashboardPage

- 项目列表优先单列卡片化。
- 项目导入、创建、继续写作等主路径按钮在 360px 宽度下可点击。

### WritingPage

- 手机首屏以编辑器主视图为中心。
- 章节列表通过 `WritingChapterListDrawer` 进入，不常驻左右分栏。
- 版本、生成、分析、上下文预览等 overlay 保持抽屉/弹窗模型，但需要满屏高度和滚动适配。
- 写作工具栏按钮组在手机端换行或分组，避免挤压正文编辑区。

### OutlinePage

- 大纲编辑、生成结果和章节列表在手机端单列排布。
- 大纲生成成功、警告、失败恢复入口必须在窄屏可见。

### PromptsPage

- 模型主配置表单移动端单列化。
- 任务预设、模型列表或复杂配置表可局部横向滚动，不做完整卡片化重构。

## 非首批页面兜底

高级调试、RAG、搜索、结构化记忆、图谱等页面先保证不产生页面级横向滚动。复杂数据表使用局部滚动容器包裹，后续再按使用频率单独优化。

## 验证设计

- 静态质量：运行 `cd frontend && npm run lint`。
- 视觉冒烟：用 390x844 和桌面宽度检查核心页面。
- 布局冒烟：可加入脚本化检查，验证核心路由 `document.documentElement.scrollWidth <= window.innerWidth`，允许标记过的局部滚动容器存在横向滚动。

## 风险与回滚

- 风险最高区域是写作页，因为它包含章节列表、编辑器、工具栏、多个抽屉和弹窗。
- 修改应优先局部增量进行，不一次性重排所有页面。
- 若某页移动端专属布局引发桌面回归，应优先回滚该页局部改动，保留全局基础设施修复。
