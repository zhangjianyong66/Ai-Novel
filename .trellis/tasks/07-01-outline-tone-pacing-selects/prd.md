# 大纲生成基调节奏快捷选择

## Goal

在“AI 生成大纲”弹窗中，将“基调”和“节奏”的常用示例提供为可快速选择的项目；用户也可以自由输入，点击生成后将本次输入保存到后端数据库，后续可从下拉候选中选择，减少重复填写并避免浏览器本地数据丢失。

## Confirmed Facts

- 前端页面位于 `frontend/src/pages/outline/OutlinePageSections.tsx`，当前“基调”和“节奏”都是普通 `<input>`。
- 表单状态类型位于 `frontend/src/pages/outline/outlineModels.ts`，字段为 `tone`、`pacing`。
- 生成请求位于 `frontend/src/pages/outline/useOutlineGenerationState.ts`，提交 payload 为 `requirements.tone` 和 `requirements.pacing`。
- 现有默认值为：
  - `tone`: `偏现实，克制但有爆点`
  - `pacing`: `前3章强钩子，中段升级，结尾反转`
- 前端已有 `<input list="...">` + `<datalist>` 的自由输入加候选提示模式。
- 后端已有 `users`、`projects`、`project_memberships` 权限模型，以及 `require_project_viewer` / `require_project_editor` 访问控制。
- 大纲相关后端路由位于 `backend/app/api/routes/outlines.py` 和 `backend/app/api/routes/outline.py`。
- 当前生成接口只接收请求体中的 `requirements.tone` 与 `requirements.pacing`，不会自动保存用户偏好。

## Requirements

- 在大纲生成弹窗中，为“基调”提供常用选项：
  - 轻松幽默，节奏明快，带一点治愈感
  - 压抑悬疑，氛围紧张，情节层层反转
  - 热血爽文，节奏快，主角一路成长打脸
  - 温柔细腻，偏日常治愈，情感推进缓慢
  - 黑暗残酷，现实感强，人物命运沉重
  - 轻松搞笑的都市爽文基调，节奏快，冲突密集，主角成长带来持续爽感
- 在大纲生成弹窗中，为“节奏”提供常用选项：
  - 快节奏，开局迅速抛出矛盾，剧情持续推进，少写日常铺垫
  - 中等节奏，主线稳定推进，穿插人物成长和感情互动
  - 慢热节奏，前期重人物关系和世界观铺垫，中后期逐渐加速
  - 张弛有度，大事件后安排短暂日常缓冲，再进入下一轮冲突
  - 中快节奏，前期快速进入主线，章节之间保持悬念和冲突，每隔数章设置一次小高潮，重要剧情后适当留出情绪缓冲。
- 用户必须仍可自由输入自定义“基调”和“节奏”。
- 用户点击生成时，当前非空的 `tone` 和 `pacing` 应保存到后端数据库。
- 下次打开同一账号、同一项目的大纲生成弹窗时，用户曾输入并生成过的“基调”和“节奏”应出现在对应下拉候选中。
- 历史配置应与内置示例合并展示并去重，避免同一文案重复出现。
- 历史配置应按当前用户和当前项目隔离，避免不同用户或不同项目之间互相污染。
- 历史配置应有数量上限，避免数据库无限增长；最近使用的项目优先保留。
- 前端打开生成弹窗时应从后端加载历史候选；加载失败时仍显示内置候选且不阻断生成。
- 保存历史失败时不应阻断大纲生成。
- 选择候选项后，应直接更新当前生成表单中的 `tone` 或 `pacing` 值。
- 保持章节数、注入世界观、注入角色卡、流式生成、生成预览等现有行为不变。
- 选项文案应集中维护，避免散落在 JSX 中。

## Acceptance Criteria

- [ ] 打开“AI 生成大纲”弹窗后，用户可以从“基调”的快捷选项中选择上述示例。
- [ ] 打开“AI 生成大纲”弹窗后，用户可以从“节奏”的快捷选项中选择上述示例。
- [ ] 用户可以继续手动输入任意“基调”和“节奏”，不受候选项限制。
- [ ] 用户点击生成后，本次非空输入会保存到当前用户、当前项目的后端历史候选。
- [ ] 用户下次打开该项目的大纲生成弹窗时，之前生成过的输入能从对应候选中选择。
- [ ] 换浏览器或清空浏览器数据后，同一账号进入同一项目仍能从后端恢复历史候选。
- [ ] 候选项去重，且最近使用的历史项优先保留。
- [ ] 用户选择快捷项后，后续生成请求仍使用选中的 `tone` 与 `pacing` 字符串。
- [ ] 未选择快捷项时，现有默认值和生成行为保持兼容。
- [ ] 后端权限检查确保只能访问有项目权限的历史候选。
- [ ] 前后端相关测试通过，至少覆盖候选项去重/保存逻辑和后端 API 行为。

## Out of Scope

- 不调整提示词模板或 LLM 输出契约。
- 不新增管理、重命名或删除历史配置的界面。
- 不把历史候选加入项目导入导出包。

## Decisions

- 保留自由输入，并使用候选下拉辅助快速选择。
- 自定义历史在用户点击“生成”时保存。
- 自定义历史保存到后端数据库，按用户和项目隔离。
- 前端保留内置示例作为后端不可用时的兜底候选，但不再以 localStorage 作为主要存储。
