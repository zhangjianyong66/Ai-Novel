# 前端移动端兼容优化

## Goal

提升 Ai-Novel 前端在手机窄屏上的可用性，优先保证核心创作流程在移动端无横向溢出、关键操作可触达、抽屉/弹窗可关闭、长文本和工具条不遮挡正文。

问题一句话：移动端用户需要能可靠进入项目、查看/编辑章节、大纲、阅读和管理常用创作资产，而不是被桌面优先的布局、固定宽度或拥挤工具条阻断。

## Confirmed Facts

- 前端位于 `frontend/`，技术栈是 React 19 + Vite 7 + TypeScript + Tailwind CSS，见 `.trellis/spec/frontend/index.md`。
- 样式规范要求优先使用 `frontend/src/index.css` 中的 `btn`、`input`、`textarea`、`select`、`ui-focus-ring`、主题 token，并避免长文本挤破容器，见 `.trellis/spec/frontend/component-guidelines.md`。
- 质量规范要求移动端长文案、按钮、toast、modal 在窄屏下不能溢出，验证优先运行 `cd frontend && npm run lint`，见 `.trellis/spec/frontend/quality-guidelines.md`。
- `frontend/src/components/layout/AppShell.tsx` 已有 `lg:hidden` 移动端导航抽屉和桌面侧栏分离；移动抽屉宽度当前为 `w-[280px]`。
- `frontend/src/components/layout/AppShell.tsx` 的页面主容器使用 `px-4 sm:px-6 lg:px-8` 和 `max-w-*`，全局页面框架已有响应式基础。
- `frontend/src/pages/writing/WritingPageSections.tsx` 是核心写作工作区；章节列表在 `lg` 以下改为抽屉，正文编辑区保留在主内容中。
- 写作页头部操作区存在多个按钮、状态徽标和 `xl:min-w-[560px]` / `xl:max-w-[720px]` 等宽度约束，窄屏风险集中在按钮换行、状态条、下拉菜单定位和编辑器空间。
- 仓库中大量页面/组件使用 `min-w-*`、固定抽屉宽度、表格 `min-w-[640px]`、`h-[calc(100vh-...)]`、`fixed bottom-*` 等模式；移动端优化需要按页面风险分层处理，而不是只改全局 CSS。
- 已有一些页面具备移动端替代形态，例如 `AdminUsersPage.tsx` 在 `md:hidden` 下展示用户卡片、写作/阅读/预览页在窄屏下用章节列表抽屉。
- 产品范围决策：首轮采用“核心创作路径优先”，先保证手机上能完成主要写作流程；高级调试、管理、RAG/Prompt 等复杂页面首轮只要求不撑破全站布局。
- `frontend/src/components/ui/Drawer.tsx` 和 `frontend/src/components/ui/Modal.tsx` 已提供基础滚动容器；移动端风险更多来自调用方传入的固定宽度/高度、底部浮动条、工具条拥挤和长文本。

## Requirements

- R1：全局应用框架在常见手机宽度下可用，顶部标题、导航按钮、账户/主题/帮助入口不横向溢出，移动端导航抽屉可滚动、可关闭、不会被安全区或长导航列表截断。
- R2：核心创作路径优先覆盖：项目首页、项目向导/设置入口、大纲页、写作页、阅读/预览页；高级调试、管理、RAG/Prompt 页面首轮只保证不造成 body 级横向溢出。
- R3：写作页移动端必须能完成章节选择、编辑标题/计划/正文/摘要、保存、AI 生成入口、章节分析入口、版本入口、状态动作和历史/上下文类抽屉关闭。
- R4：弹窗、抽屉、浮动卡片和底部下一步栏在手机宽度下不得遮挡主要操作；内容高度超过视口时必须可滚动。
- R5：表格或调试型页面允许横向滚动，但必须放在明确的滚动容器内，不能撑破整个页面造成全站横向滚动。
- R6：长中文文案、项目名、章节标题、request id、模型名、任务类型等动态文本必须截断、换行或断词，不能压缩按钮或撑破卡片。
- R7：保持现有桌面布局语义和操作能力，不为移动端改动破坏桌面侧栏、桌面多栏工作区或既有交互。
- R8：优先使用现有 Tailwind token、基础 UI 类和组件结构；新增通用响应式辅助样式必须有明确复用价值。

## Acceptance Criteria

- [ ] 在 375px 宽度手机视口下，应用主框架和优先覆盖页面没有 body 级横向滚动。
- [ ] 移动端导航抽屉内容可完整访问，项目导航较长时可滚动，点击导航项后能关闭。
- [ ] 写作页在 375px 宽度下可完成章节列表打开/选择、正文编辑、保存/状态动作、AI 生成入口、章节分析入口和版本入口，不出现不可触达按钮。
- [ ] 写作页所有主要抽屉/弹窗在手机视口下可关闭、内容可滚动，底部或浮动元素不永久遮挡正文编辑。
- [ ] 大纲页、预览/阅读页、首页至少完成一轮移动端布局检查并修复发现的阻断问题。
- [ ] 对必须保留宽表格的页面，移动端横向滚动限定在表格容器内。
- [ ] `cd frontend && npm run lint` 通过。
- [ ] 如改动抽取了纯函数、状态规则或复杂响应式判定，补充相应 Vitest；纯样式调整不强制新增测试。

## Out of Scope

- 不重做整套视觉设计或主题色。
- 不新增移动端原生 App、PWA 离线能力或手势导航体系。
- 不改变后端 API、数据库结构或 LLM 调用流程。
- 不承诺所有高级调试/管理页面达到完整移动端一屏内体验；这些页面至少不能破坏全站布局。
